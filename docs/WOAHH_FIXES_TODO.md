- [ ] **28. "Reload/refresh" error when opening pages (Promotions et al.) — ROOT CAUSE FOUND, fix pending.** What's happening: the dashboard code-splits every page (46 `lazy()` routes in `App.tsx`). Each deploy to `main` gives chunk files new hashed names — any browser tab opened BEFORE the deploy still runs the old router, and the first page it tries to lazy-load after the deploy 404s (old chunk gone) → the ErrorBoundary shows "Something went wrong — Refresh the page". After refresh the new HTML+chunks load, so it works. It'll hit whichever page is clicked first after any deploy — Promotions today, anything tomorrow; we deploy often, so merchants see it a lot. **Fix (small, ~20 lines):** (a) listen for Vite's built-in `vite:preloadError` event in `main.tsx` → `preventDefault()` + auto-reload once (sessionStorage guard against reload loops); (b) make the ErrorBoundary recognise chunk-load failures and silently reload once instead of showing the scary screen; (c) optional polish: a "new version available" toast via build-id check instead of hard reload mid-task. No code currently handles this (verified: zero `vite:preloadError`/ChunkLoadError handling in `src/`). *(Added 2026-06-12, diagnosed same day.)*
